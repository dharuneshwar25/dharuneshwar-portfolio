Smart Solar Panel Monitoring Drone
Executive Summary

Development has successfully established a robust Geo-spatial Processing Pipeline and a secure Drone Management System with OTP verification. The multi-tier architecture now supports reliable GeoTIFF management, accurate coordinate extraction via Python services, and comprehensive CRUD operations, laying the groundwork for upcoming visual map integrations.

Technical Architecture and Stack

The application utilizes a high-performance, modular stack. The Frontend uses the MERN stack (React.js) with tools like Styled-Components and jsPDF for interactive reporting. The Backend relies on Node.js/Express and Sequelize for API routing and persistence, while a specialized Geo-Processor built on Python (FastAPI) handles complex calculations using libraries such as Rasterio, Pyproj, and Shapely. Data is securely managed in a MySQL database.

Completed Deliverables
  • Core features implemented include Drone Data Management, providing full backend and frontend capabilities for creating and deleting records with automatic file system cleanup.
  • The Geo-Spatial Pipeline is fully established, linking the Node.js API to the Python processor to convert pixel coordinates to WGS84 Latitude/Longitude and generate GeoJSON buffers.
  • Additionally, Multi-Factor Authentication for password resets and System Status Modules (battery, connectivity, flight readiness) are fully functional.
Remaining Scope and Future Integration

The final phase focuses on visualization and external data. A critical pending feature is the TIF-to-JPEG conversion service to enable browser display of GeoTIFFs, followed by Virtual Map Integration (e.g., Leaflet) to render boundaries and data points. Remaining tasks also include integrating the Weatherstack API for real-time weather data and developing a visual interface to display Solar Panel Health statuses.

ICU Vital Monitoring System
Executive Summary

Development has successfully concluded on the Android-based client for the ICU Vital Monitoring System. The application establishes a low-latency connection using Firebase to visualize critical patient health data in real-time. Key achievements include the successful implementation of unique device pairing logic and a high-priority alert system that notifies medical staff immediately when vitals deviate from normal physiological ranges.

Technical Architecture and Stack

The solution is built on a native Android architecture designed for reliability and speed in medical settings.

  • Mobile Environment: Developed using Android Studio, ensuring full access to native device notification and audio APIs.
  • Real-Time Engine: Utilizes Firebase Realtime Database as the bridge between the hardware and the app. This ensures that data pushed by the ESP32 sensors (via Wi-Fi) is instantly reflected on the doctor's screen without manual refreshing.
  • Data Structure: The architecture relies on Unique Device IDs to segregate and securely track data streams for individual patients.
Completed Deliverables
  • Real-Time Vitals Dashboard: The application successfully renders live data streams for Blood Oxygen (SpO2), ECG (Electrocardiogram), Body Temperature, and Heart Rate.
  • Automated Alert Logic: A critical safety feature is fully operational. The app continuously monitors incoming data against pre-defined medical thresholds. If a value drops below or exceeds the normal range, the system triggers an immediate audio-visual alarm to alert the attending doctor.
  • IoT Device Linking: The software interface for the ESP32 hardware is complete. The app creates a dedicated listener for specific Device IDs to ensure accurate patient-to-device mapping.
Scope and Integration Boundaries
  • Software-Hardware Interface: This project scope focused exclusively on the mobile application development.
  • Integration Readiness: While the hardware (ESP32/Sensors) was developed separately, the application's communication protocols are fully established and tested to receive and parse JSON data payloads sent from the hardware units via the cloud.

Mobile Application Developer Intern - Aquila Innovations
Project: GREET Tourism App

Over a 4-week period, I developed the cross-platform mobile application GREET using .NET MAUI. The app allows tourists to scan monuments to instantly retrieve detailed information, demonstrating skill in native feature integration and cross-platform application development.

Executive Summary

Development efforts have centered on the "History" module, a core component of the broader GREET tourism ecosystem. Built using .NET MAUI, this cross-platform module enhances the tourist experience by enabling real-time monument scanning and information retrieval. Simultaneously, a dedicated Admin side has been deployed to manage rich media content, specifically 360° video tours, ensuring immersive storytelling for users.

Technical Architecture and Stack

The module leverages .NET MAUI to ensure a unified code base for both Android and iOS platforms. Key technical components include:

  • Mobile Client: .NET MAUI app enabling real-time monument scanning through the device camera.
  • Video Processing Engine: Admin uploads a 360° monument video, from which the system extracts all frames and generates RGB-based image fingerprints.
  • Recognition Pipeline: Client-captured images are uploaded and matched against the fingerprint database to accurately identify the monument.
  • Integration: Architected as a plug-and-play module that integrates smoothly with the existing GREET super-app ecosystem.
Completed Deliverables
  • Tourist Interface (Scanner): The "Scan-to-Learn" feature is fully functional. Tourists can point their device at a popular monument to instantly trigger the retrieval of historical data, facts, and significance.
  • Admin Content Management: A robust admin interface is operational, allowing administrators to upload and manage monument details.
  • 360° Video Integration: The video player module has been successfully implemented, enabling admins to post immersive 360° footage of locations, which is then rendered correctly on the user's device.
Integration with GREET Ecosystem

While the "History" module is functional as a standalone feature, the next phase focuses on deep integration with the wider GREET application features:

  • Tour Planning Connection: Linking scanned monuments to the Tour Planning module to suggest nearby sites automatically.
  • Booking Integration: Connecting historical sites with the Hotel Booking and Offline Payment systems to facilitate ticket purchases or stays near the scanned monument.
Skills Learned

.NET MAUI framework, cross-platform development, camera integration, 360° media rendering, modular application architecture.
